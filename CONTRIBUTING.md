# Contributing

Thank you for your interest in contributing to **bun-workspaces**.

Contributions are welcome. Well-suited contributions include:

- Bug fixes
- Documentation corrections or clarifications
- Small, focused improvements

Please ensure all CI checks pass before requesting a review. Pull requests should automatically run the required checks.

The project is actively developed with a clear [roadmap](https://bunworkspaces.com/roadmap).

If you have larger ideas than the suggested contributions above, consider heading to our [discussions](https://github.com/bun-workspaces/bun-workspaces/discussions) to see if others have posted similar ideas or to get feedback first.

Some local commands you may run first to help with checks:

- `bun format` - run Prettier
- `bun type-check` - check the project's type safety
- `bun lint` - lint the project
- `bun bw:test` - run tests for the bun-workspaces package

The project's [CLAUDE.md](./CLAUDE.md) can actually serve as a good overview of the project's development practices and coding style.
