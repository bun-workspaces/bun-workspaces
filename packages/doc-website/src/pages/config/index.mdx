# Configuration > General

<ConfigHeader activeHref="home" divider/>

Configuration is set by JSON. Config files can be written in TypeScript, JavaScript, JSON, or JSONC.

See [Root Configuration](/config/root) and [Workspace Configuration](/config/workspace) for more information on the purpose of each field.

## Quick Examples

### Root Configuration

The [root configuration](/config/root) is for project-wide settings.

A root configuration in JSON, at <code>path/to/your/project/{createConfigLocationPath("jsonFile", ROOT_CONFIG_FILE_NAME, ROOT_CONFIG_PACKAGE_JSON_KEY)}</code>:
<SyntaxHighlighter language="json">
  {JSON.stringify((exampleRootConfigSimple1), null, 2)}
</SyntaxHighlighter>

A root configuration in TypeScript, at <code>path/to/your/project/{createConfigLocationPath("tsFile", ROOT_CONFIG_FILE_NAME, ROOT_CONFIG_PACKAGE_JSON_KEY)}</code>:
<SyntaxHighlighter language="typescript">

### Workspace Configuration

[Workspace configurations](/config/workspace) are defined in the directory of the workspace.


A workspace configuration in JSON, at <code>path/to/project/workspace/{createConfigLocationPath("jsonFile", WORKSPACE_CONFIG_FILE_NAME, WORKSPACE_CONFIG_PACKAGE_JSON_KEY)}</code>:

## Precedence (Resolution Order)

### Configuration locations

Configuration is read from the following locations, in order of precedence:
<ol className="config-location-list">
{CONFIG_LOCATION_TYPES.map(type => <li key={type}><code>{createConfigLocationPath(type, "bw.<config name>", "<key>")}</code></li>)}
</ol>

The first configuration location found will be used.

### Value Resolution

Values related to configuration are resolved in the following order of precedence:
1. An explicit CLI or API argument/option passed to a method/function/command
2. A value provided in a configuration file
3. A value provided via an [environment variable](/config/env-vars), if applicable
4. The generic default value, if applicable
